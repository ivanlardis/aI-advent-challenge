# Рекомендация: для чего хватает локальной модели

**Железо:** MacBook Pro M3 Max, 48 GB RAM.
**Стек:** PyCharm Community + Continue.dev + Ollama.
**Промпт-правила:** `CLAUDE.md` целиком в `systemMessage` для чат-моделей (см. `.continue/config.yaml`).
**Параметры для кода:** `temperature: 0.2`, `topP: 0.9`, `contextLength: 8192`.

## TL;DR

Локалка — ассистент для **черновой работы и автономных сценариев**. Фичи и агентные bug-fix я пока оставляю на cloud.

## Где локальная модель **достаточна**

| Сценарий | Почему годится | Какую модель брать |
|----------|---------------|---------------------|
| **Автокомплит коротких строк** (имя переменной, конец выражения) | Не нужно понимать архитектуру, FIM делает узкую задачу | qwen2.5-coder:7b-base (FIM) |
| **Boilerplate** (docstring, try/except, простой pytest-скелет) | Шаблоны, модель копирует паттерны из контекста | qwen2.5-coder:7b / :14b |
| **Переименования и мелкий рефактор** (extract variable, rename) | Нужна локальная правка одной функции | qwen2.5-coder:7b |
| **Работа в самолёте / без интернета / под NDA** | Cloud отпадает по определению | qwen2.5-coder:14b |
| **Массовые однотипные правки** (100 файлов → одинаковая замена) | Cloud лимиты и цена, локалка крутится бесплатно | qwen2.5-coder:14b |
| **Черновик тестов к чистому модулю** | Нет завязки на фреймворк, только сигнатуры | qwen2.5-coder:7b |

## Где локальная модель **не заменяет cloud**

| Сценарий | Почему нет | Что делать |
|----------|-----------|------------|
| **Интеграция с незнакомой библиотекой** (Chainlit, LangChain) | Маленькие модели выдумывают API (`@cl.page`, `cl.append_message`, `@app.route` в Chainlit) | Cloud (Sonnet/Opus) — у них свежее знание API |
| **Агентный bug-fix по смутному описанию** | В тесте `/summary` ни одна локалка не нашла рассинхрон ключей `"total"` vs `"total_tokens"` | Cloud + `.claude/agents/bug-fix.md` профиль |
| **Большой контекст (>8K) + многофайловые изменения** | Маленькие локалки теряют нить, зацикливаются (gemma3:1b сожгла 80K токенов на повторе строки) | Cloud (Claude с 200K контекста) |
| **Production-код, который пойдёт в PR без ревью** | Качество недостаточно стабильное, нужны правки | Cloud |

## Лучшая связка под это железо (target-конфиг)

После апгрейда интернета и загрузки coder-моделей:

- **Автокомплит:** `qwen2.5-coder:7b-base` (FIM), `temperature: 0.1`, `contextLength: 4096`
- **Чат / edit / apply:** `qwen2.5-coder:14b`, `temperature: 0.2`, `topP: 0.9`, `contextLength: 8192`, полный `CLAUDE.md` в systemMessage
- **Быстрый draft:** `qwen2.5-coder:7b` (тот же config, но быстрее в 1.5-2×)
- **Альтернатива для нетривиальных задач:** `gemma3:12b` (проверить — на 1b-версии семейство показало себя плохо)

## Честный вывод из реального прогона

На доступных **generalist-моделях ≤3B** (qwen2.5:3b, gemma3:1b, qwen2.5:0.5b) локальный ассистент **не заменяет cloud** даже на простой задаче добавления `/history` команды — ни одна модель не выдала рабочий код с первого раза.

Это предсказуемо: HumanEval у generalist 3B ~57%, у coder 14B — 89%. Разрыв огромный. Пока не загружу coder-модели, на локалке делаю только автокомплит и boilerplate; всё что требует понимания Chainlit/LangChain — cloud.
