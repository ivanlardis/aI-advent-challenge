# Рекомендация: для чего хватает локальной модели

**Железо:** MacBook Pro M3 Max, 48 GB RAM.
**Стек:** PyCharm Community + Continue.dev + Ollama.
**Основная модель:** `gemma4:e4b` (Gemma 4 E4B, 9.6 GB, generalist с упором на agentic+coding).
**Промпт-правила:** `CLAUDE.md` целиком в `systemMessage` для чат-моделей (см. `.continue/config.yaml`).
**Параметры для кода:** `temperature: 0.2`, `topP: 0.9`, `contextLength: 16384`.

## TL;DR

После апгрейда с generalist ≤3B на **Gemma 4 E4B** локалка закрывает большинство повседневных задач, включая агентный bug-fix по смутному описанию. Cloud остаётся для задач с незнакомыми фреймворками, больших рефакторингов и кода, идущего в PR без ревью.

## Где локальная модель **достаточна** (Gemma 4 E4B)

| Сценарий | Почему годится | Подтверждение |
|----------|---------------|---------------|
| **Агентный bug-fix** (формат «Что нашёл / Что починить / Что проверить») | Gemma 4 E4B нашла рассинхрон ключей `"total"` vs `"total_tokens"` с первой попытки — на уровне Claude Sonnet | `task-runs/day2-gemma4_e4b.md` |
| **Фича по паттерну проекта** (новая команда в `app.py`) | Правильная структура хендлера, `cl.user_session`, регистрация по паттерну CLAUDE.md. Ошибка только в одной сигнатуре (`cl.ChatState` — которого нет), правится одной строкой | `task-runs/day1-gemma4_e4b.md` |
| **Автокомплит коротких строк** | Не нужно понимать архитектуру, модель берёт локальный контекст | конфиг `.continue/config.yaml` |
| **Boilerplate** (docstring, try/except, pytest-скелет) | Шаблоны, модель копирует паттерны из контекста | тестовые прогоны |
| **Переименования и мелкий рефактор** | Нужна локальная правка одной функции | |
| **Работа в самолёте / без интернета / под NDA** | Cloud отпадает по определению | |
| **Массовые однотипные правки** (100 файлов) | Cloud лимиты и цена, локалка крутится бесплатно | |

## Где локальная модель **не заменяет cloud**

| Сценарий | Почему нет | Что делать |
|----------|-----------|------------|
| **Интеграция с незнакомой/экзотической библиотекой** | На свежих/редких фреймворках даже Gemma 4 может выдумать API (как выдумала `cl.ChatState` для Chainlit) | Cloud (Sonnet/Opus) — свежее знание API |
| **Большой многофайловый рефакторинг** | Не тестировалось; в Chainlit `app.py` масштаб невелик, а на больших проектах контекстное окно 16K становится узким | Cloud (Claude с 200K) |
| **Production-код, который пойдёт в PR без ревью** | Качество достаточно, но на одной сигнатуре Gemma 4 всё равно ошиблась. Для PR без ревью — слишком рискованно | Cloud, либо локалка + ручной ревью |

## Конфиг моделей

**Основной чат / edit / apply:** `gemma4:e4b` — temperature 0.2, topP 0.9, contextLength 16384, полный `CLAUDE.md` в systemMessage.
**Baseline для сравнений:** `qwen2.5:3b` — оставлен для быстрых прогонов и демонстрации разницы.

Coder-специализированные модели (`qwen2.5-coder:7b`, `qwen2.5-coder:7b-base`) в target-секции `.continue/config.yaml` закомментированы — Gemma 4 E4B покрывает те же задачи при одной модели вместо связки из трёх.

## Главный вывод из двух прогонов

**Скачок между generalist ≤3B и Gemma 4 E4B качественный, а не линейный.** Где qwen2.5:3b выдумывал FastAPI и гадал про природу бага, Gemma 4 E4B находит настоящую причину рассинхрона ключей и выдаёт точный diff. Это первая локалка, которую я могу доверить реальной bug-fix задаче на этом проекте.

Скорость 66 tok/s на M3 Max — медленнее, чем у 3B-моделей (120-285 tok/s), но по subjectивному ощущению от Continue.dev ответ приходит быстро, а главное — по делу. «Быстрый мусор» бесполезен в цикле разработки, где каждая правка требует перезапуска теста.
